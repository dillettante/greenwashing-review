# Greenwashing Counsel — 현행 구축계획

기준일: 2026-07-05

## 1. 운영 목표

지속가능경영보고서·광고·홍보자료에서 환경주장을 추출하고, 한국법상 표시·광고 해당성·오인 가능성·실증 수준을 평가한 뒤 변호사 승인 하에 신고·조사요청·고발 문서 초안을 생성한다. 한국법만 직접 판단근거로 사용하고 외국 규범은 비교법적 보강으로 분리한다.

## 1-A. 3계층 구조 (핵심 설계)

법적 판단은 정규식이 하지 않는다. 판단은 변호사+Claude 세션이 korean-law MCP로 한다.

```
① 추출·트리아지 (CLI, 정규식)   → 1-shortlist.json + 기계 1차분류(참고, 법적 판단 아님)
② 정밀 법적 평가 (Claude 세션)  → korean-law MCP로 조·항·호 포섭·심결례·오인가능성 → 2-evaluation.json
③ 문서 생성·게이트 (CLI)        → 2-evaluation.json 병합 → 보고서·검토표 → 승인 → 고발장
```

step ② 절차·스키마는 [EVALUATION-SOP.md](EVALUATION-SOP.md)에 고정. `assess`를 다시 실행하면
`2-evaluation.json`이 보고서·고발장에 병합된다. 정밀평가 없는 주장은 보고서에 `[미평가]`로 표시되어 숨겨지지 않는다.

## 2. 현재 구현 상태

### 완료

- 국내 핵심 규범 6개를 국가법령정보센터 공식 원문에서 수집해 버전·조문·원문/본문/조문 해시 단위로 저장
- 로컬 조문 DB가 없으면 사건 평가를 중단하는 법률근거 게이트
- PDF·DOCX·PPTX·HTML·이미지 입력, 페이지별 주장 추출, 광고성·유형·기계 위험점수
- 위험점수 재보정: 실증 부재만으로 전 주장을 높음으로 밀지 않도록 misleading 가중 제거. 예시 기업 117건 분포가 매우높음 9·높음 45·중간 63으로 실제 트리아지 가능
- 정밀평가 seam: `1-shortlist.json`(정밀평가 대상 최대 20건)과 `2-evaluation.json`(세션 산출) 병합. 보고서에 `법적 평가(변호사·korean-law MCP)` 블록과 `[미평가]` 표시 분리
- korean-law MCP로 표시광고법 제3조·제5조 현행 원문 확인·인용(실증)
- 회사 홈페이지·직접 지정 URL·Google News RSS를 통한 공개자료 교차확인 루프
- 공개 보도와 기업 실증자료의 분리: 공개자료가 일치해도 실증 부족 점수를 자동 감경하지 않음
- 산출물 `.md`(법률검토보고서=제출경로 통합) + `.xlsx`(주장검토표·증거목록 작업본), 승인 후 제출문서(.md) 초안. 각 주장에 실증·검증(웹 리서치) 블록 렌더
- 공식 규범 변경 감시, 국가법령정보센터 판례 후보 자동 수집, 공정위·기후에너지환경부 검색결과 변경 감시
- 판례·처분 후보의 `pending_human_review` 검증 큐와 수록·제외·중복 상태 기록

### 진행 중

- **심결례·판례 검증 경로 (후속 과제, 미착수)** — 법제처 API(korean-law MCP) ftc·precedent 도메인이 그린워싱 사례를 거의 반환하지 않음. 대안: ① 공정위 사건검색(case.ftc.go.kr) DB화 또는 Open API 개방 대기, ②  사내 판례·심결례 DB. 상세는 MAINTENANCE.md. 착수 전까지 제출 문서는 조문 근거만으로 작성
- 추출 오탐 개선: 정책·목차 나열형 문단(예: '환경관리 정책 생물다양성…')이 주장으로 잡히는 blob 제거
- 미국·EU·영국 핵심 규범 원문·사례의 동일한 버전 DB 전환
- 이미지·도안·색상·인증마크의 좌표 단위 자동 판독
- 변호사 정답 세트 3~5건을 이용한 추출 재현율·등급 일치율 측정

## 3. 사건 처리 루프

1. `context.yaml`, 검토대상, 기업 실증자료를 사건 폴더에 배치
2. **①추출·트리아지**: `assess`로 주장 추출·기계 위험점수·`1-shortlist.json`·`1-worklist.md` 생성
3. 회사 홈페이지·언론·직접 지정 URL 검색 및 원문/검색포인터 스냅숏(`--with-public-check`)
4. **②정밀평가**: Claude 세션에서 shortlist를 korean-law MCP로 포섭·심결례 확인 + **웹 리서치로 회사 주장 지표·인증의 실재·범위·반증 검증**(위험 확정 전) → `2-evaluation.json` (EVALUATION-SOP.md)
5. **③병합·생성**: `assess` 재실행으로 `2-evaluation.json`을 보고서(.md)·고발장(.md)에 병합
6. 사람이 인증서·LCA·원자료·기사 원문과 반증자료를 검증
7. 변호사 승인(`approve`) 후 제출문서 초안(`draft`) 생성, `verify`로 무결성 확인

공개 보도는 실증자료가 아니므로 점수 자동 감경에 쓰지 않는다.

기본 실행:

```bash
python3 -m greenwashing assess matters/example --mode public --with-public-check
# → 1-shortlist.json 확인 → 세션에서 2-evaluation.json 작성(EVALUATION-SOP.md) →
python3 -m greenwashing assess matters/example --mode public   # 2-evaluation.json 병합
python3 -m greenwashing verify matter-id
```

## 4. 지식 DB 최신화 루프

- 매주: 국내 6개 핵심 규범 현행 원문 재수집, 시행일·본문 해시 비교, 판례 검색식 실행
- 매월: 공정위·기후에너지환경부·국가법령정보센터 저장 검색의 결과 링크 집합 비교
- 사건 제출 전: `pre_filing` 감시, 7일 이내 현행성 재확인, 판례 후보와 후속절차 수동 검증
- 분기: coverage registry의 검색식·기간·제외·중복 사유와 외국법 상태 검토

판례·처분 검색 결과는 자동으로 법적 근거 DB에 들어가지 않는다. 공식 원문, 사건번호, 절차단계, 확정 여부, 후속절차를 사람이 확인한 뒤 구조화 JSON으로 `corpus import-json`해야 한다.

## 5. 다음 완료 기준

- 한국 처분·판례 coverage의 검색기간과 검색식이 고정되고 수록·제외·중복 사유가 모두 남을 것
- 예시 기업 외 실제 자료 2~4건으로 중요주장 재현율 95%와 고위험 누락 0건을 검증할 것
- 공개자료 루프의 검색 누락·중복·기사 원문 미확보율을 측정할 것
- 외국 핵심 규범도 공식 원문·버전·해시를 보유하고 한국 직접근거와 출력상 분리될 것
